## Network flags

The `serve` command accepts these network flags:

| Flag | Default | Description |
| --- | --- | --- |
| `--port` | `0` | Port to listen on |
| `--hostname` | `127.0.0.1` | Hostname to bind |
| `--cors <origin>` | `[]` | Additional CORS origins (repeatable) |

## Port behavior

When `--port 0` (the default):
1. Try port 4096
2. If 4096 is busy, pick a random available port

When a specific port is given, the server fails if that port is unavailable.

## Resolution priority

CLI flags override config file values:

1. `--port` flag
2. `server.port` in `~/.config/0x0/config.json`
3. Default (`0`)

Same for `--hostname` and `--cors`.

## CORS

Requests from `localhost` and `127.0.0.1` are always allowed. Additional origins can be added via `--cors` or `server.cors` in config.

## Examples

```bash
# Default: localhost:4096
0x0

# Fixed port, all interfaces
0x0 --hostname 0.0.0.0 --port 4096

# Allow a specific browser origin
0x0 --port 4096 --cors https://my-app.example.com
```

## mDNS service discovery

The server can advertise itself on the local network via mDNS (Bonjour/Zeroconf), allowing clients to discover it automatically without knowing the port.

Enable in config:

```json
{
  "server": {
    "mdns": true,
    "mdnsDomain": "my-machine"
  }
}
```

| Field | Default | Description |
| --- | --- | --- |
| `server.mdns` | `false` | Enable mDNS advertisement |
| `server.mdnsDomain` | hostname | Custom domain name for the mDNS service |
