## Global options

Available on all commands:

- `-h`, `--help`: show command help
- `-v`, `--version`: print installed version
- `--print-logs`: print runtime logs to `stderr`
- `--log-level DEBUG|INFO|WARN|ERROR`: set log level

## `0x0` (default: serve)

Start the completion server. This is the default command.

```bash
0x0
0x0 serve
0x0 --port 4096 --hostname 0.0.0.0
```

Options:

- `--port` (default `0` -- tries 4096, then random)
- `--hostname` (default `127.0.0.1`)
- `--cors <origin>` (repeatable)

## `0x0 commit-msg`

Generate a commit message from staged changes.

```bash
0x0 commit-msg
0x0 commit-msg --provider claude --model claude-sonnet-4-6
0x0 commit-msg --verbose
```

Options:

- `--provider`: `claude` or `codex` (default: auto-detect)
- `--model`: model ID (default: per-provider)
- `--verbose`: enable debug output

## `0x0 hook <install|uninstall>`

Manage the `prepare-commit-msg` git hook.

```bash
0x0 hook install
0x0 hook uninstall
```

## `0x0 completion`

Output shell completion script:

```bash
# bash / zsh
eval "$(0x0 completion)"
```

## Network flags

Shared by the `serve` command:

| Flag | Default | Description |
| --- | --- | --- |
| `--port` | `0` | Port to listen on (0 = try 4096, then random) |
| `--hostname` | `127.0.0.1` | Hostname to bind |
| `--cors` | `[]` | Additional CORS origins (repeatable) |

Config file values (`server.port`, `server.hostname`, `server.cors`) are used as defaults. CLI flags override config.
