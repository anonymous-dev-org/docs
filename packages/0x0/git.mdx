Generate conventional commit messages from your staged changes. Built into the `0x0` binary -- no separate package needed.

## Usage

```bash
# Stage your changes
git add .

# Generate a commit message (auto-detects provider)
0x0 commit-msg

# Specify provider and model
0x0 commit-msg --provider claude --model claude-sonnet-4-6
0x0 commit-msg --provider codex --model o4-mini

# Debug output
0x0 commit-msg --verbose
```

## Install as git hook

```bash
# Install prepare-commit-msg hook
0x0 hook install

# Remove the hook
0x0 hook uninstall
```

Once installed, every `git commit` (without `-m`) automatically generates a message. The hook:
- Preserves existing hook content
- Uses markers for safe uninstall
- Skips when a message is already provided (`-m`, merge, squash, amend)
- Respects `core.hooksPath` (husky compatible)

## How it works

1. Reads staged changes via `git diff --cached` (truncated at 20K chars)
2. Ensures the 0x0 server is running (auto-starts if needed)
3. Sends the diff to `POST /completion/text` with a commit message prompt
4. Outputs the result in Conventional Commits format

## Configuration

| Source | Provider | Model | URL | Auth |
| --- | --- | --- | --- | --- |
| CLI flags | `--provider` | `--model` | -- | -- |
| Environment | `GIT_AI_PROVIDER` | `GIT_AI_MODEL` | `GIT_AI_URL` | `GIT_AI_AUTH` |
| Auto-detect | Prefers `claude`, fallback `codex` | Per-provider default | `http://localhost:4096` | none |

### Environment variables

| Variable | Description |
| --- | --- |
| `GIT_AI_PROVIDER` | `claude` or `codex` |
| `GIT_AI_MODEL` | Model ID override |
| `GIT_AI_URL` | Server URL (default: `http://localhost:4096`) |
| `GIT_AI_AUTH` | Basic auth as `username:password` |
| `GIT_AI_DEBUG` | Set to `1` for debug output |

## Commit format

Generated messages follow [Conventional Commits](https://www.conventionalcommits.org/):

```
<type>(<scope>): <description>
```

Types: `feat`, `fix`, `refactor`, `docs`, `style`, `test`, `chore`, `perf`, `ci`, `build`.
