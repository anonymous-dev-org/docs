Inline ghost text code suggestions for Neovim, similar to GitHub Copilot. Routes completions through the 0x0 server, which handles provider routing, project context gathering, and learning from your accept/reject patterns.

Uses `claude-haiku-4-5-20251001` by default for fast time-to-first-token (~200-400ms).

## Requirements

- Neovim >= 0.10
- `curl` in PATH
- 0x0 server running (the plugin can auto-start it)
- At least one provider: `claude` CLI on PATH, or `OPENAI_API_KEY` set

## Installation

### lazy.nvim

```lua
{
  "anonymous-dev-org/0x0-completion.nvim",
  event = "InsertEnter",
  opts = {},
}
```

### packer.nvim

```lua
use {
  "anonymous-dev-org/0x0-completion.nvim",
  config = function()
    require("zeroxzero-completion").setup()
  end,
}
```

## Configuration

```lua
require("zeroxzero-completion").setup({
  server_url = nil,                          -- auto-detected (http://localhost:4096)
  auth = nil,                                -- optional {username = "...", password = "..."}
  model = "claude-haiku-4-5-20251001",       -- model for completions
  provider = nil,                            -- "claude" or "codex" (auto-detect)
  max_tokens = 256,                          -- max completion length
  debounce_ms = 150,                         -- wait before requesting
  max_prefix_lines = 100,                    -- context before cursor
  max_suffix_lines = 50,                     -- context after cursor
  cache_size = 64,                           -- LRU cache entries
  project_root = nil,                        -- auto-detected from git root
  disabled_filetypes = {
    TelescopePrompt = true,
    NvimTree = true,
    lazy = true,
    mason = true,
    help = true,
    [""] = true,
  },
  keymaps = {
    accept = "<Tab>",                        -- accept full completion
    dismiss = "<C-]>",                       -- dismiss ghost text
    accept_word = "<M-w>",                   -- accept first word
    accept_line = "<M-l>",                   -- accept first line
    toggle = "<M-Bslash>",                   -- toggle on/off (normal mode)
  },
})
```

## How it works

1. Monitors `TextChangedI` events for cursor changes
2. Extracts prefix and suffix with language detection
3. Debounces requests (150ms default)
4. Sends `POST /completion` with `project_root` and `filename` -- the server gathers all context
5. Streams the response and renders ghost text via extmarks
6. On accept (`<Tab>`), calls `POST /completion/accept` -- the server learns
7. On dismiss (`<C-]>`), calls `POST /completion/reject` -- negative signal

The server handles everything -- convention analysis, import resolution, symbol lookup, memory retrieval, and rule injection. The plugin just sends prefix/suffix/filename/project_root.

## Project context

When `project_root` is set (auto-detected from git root by default), every completion request benefits from:

- **Conventions** -- formatting style, naming, import patterns detected from the codebase
- **Imports** -- exported APIs from files the current file imports
- **Siblings** -- structural patterns from neighboring files
- **Types** -- definitions of types referenced near the cursor
- **Memory** -- past accepted completions and learned rules

This means completions match your project's style from the first keystroke, and improve over time as you accept/reject suggestions.

## Keymaps

| Keymap | Mode | Action |
| --- | --- | --- |
| `<Tab>` | Insert | Accept the full completion (falls through when no ghost text) |
| `<C-]>` | Insert | Dismiss current ghost text |
| `<M-w>` | Insert | Accept first word |
| `<M-l>` | Insert | Accept first line |
| `<M-\>` | Normal | Toggle completions on/off |

## Commands

| Command | Description |
| --- | --- |
| `:ZeroCompletionToggle` | Toggle autocompletion on or off |
| `:ZeroCompletionClear` | Clear current ghost text |
