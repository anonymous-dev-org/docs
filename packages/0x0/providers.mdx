0x0 supports two LLM providers for completions and text generation.

## Claude

Uses the `claude` CLI binary to stream completions. No API key management needed -- authentication is handled by the Claude CLI itself.

**Detection:** `claude` binary on `PATH`.

| Model | Context | Output | Notes |
| --- | --- | --- | --- |
| `claude-haiku-4-5-20251001` | 200K | 8K | Default for completions (fast) |
| `claude-sonnet-4-6` | 200K | 64K | Higher quality |
| `claude-opus-4-6` | 200K | 32K | Highest quality |

## Codex (OpenAI)

Uses the OpenAI API directly via `OPENAI_API_KEY`. No SDK dependency -- just HTTP requests.

**Detection:** `OPENAI_API_KEY` environment variable is set.

| Model | Context | Output | Notes |
| --- | --- | --- | --- |
| `o4-mini` | 200K | 32K | Default for completions (fast, reasoning) |
| `o3` | 200K | 32K | Reasoning |
| `gpt-5-codex` | 400K | 128K | Large context |

## Provider selection

The server auto-detects available providers. When both are available, Claude is preferred.

You can override per-request:

```json
{
  "prefix": "...",
  "suffix": "...",
  "provider": "codex",
  "model": "o4-mini"
}
```

Or set a default in config:

```json
{
  "completion": {
    "default_provider": "codex",
    "default_model": "o4-mini"
  }
}
```
