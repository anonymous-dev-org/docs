0x0 delegates LLM inference to external provider SDKs. Each provider requires a corresponding CLI
binary on your `PATH` for authentication and availability detection, but actual inference runs
through the provider's SDK (Claude Agent SDK for Claude Code, JSON-RPC app server for Codex).

## Built-in providers

| Provider | ID | CLI binary | Models |
| --- | --- | --- | --- |
| Claude Code | `claude-code` | `claude` | `claude-sonnet-4-6` (200K context, 64K output), `claude-opus-4-6` (200K context, 32K output), `claude-haiku-4-5-20251001` (200K context, 8K output) |
| Codex | `codex` | `codex` | `gpt-5-codex` (400K context: 272K input, 128K output; reasoning), `o3` (200K context, 32K output; reasoning), `o4-mini` (200K context, 32K output; reasoning) |

Only providers whose CLI binary is found on `PATH` appear in `0x0 models` and the model picker.

Additional providers (Copilot, GitLab, Antigravity) can be made available through
[plugins](/packages/0x0/plugins) that register `auth` hooks.

## Provider config

Configure providers in `config.yaml` under the `provider` key:

```yaml
provider:
  claude-code:
    options:
      timeout: 300000
    models:
      claude-sonnet-4-6:
        variants:
          low: {}
          medium: {}
          high: {}
```

### Provider options

| Field | Type | Default | Description |
| --- | --- | --- | --- |
| `apiKey` | `string` | -- | API key (supports `{env:VAR}` interpolation) |
| `baseURL` | `string` | -- | Base URL override for API requests |
| `enterpriseUrl` | `string` | -- | GitHub Enterprise URL (Copilot provider only) |
| `timeout` | `number \| false` | `300000` | Request timeout in ms, or `false` to disable |

### Model configuration

- `models`: per-model configuration including `variants` (reasoning effort levels, can be disabled with `disabled: true`)

## Provider config files

Provider-specific config can be stored in separate files:

- **Global**: `~/.config/0x0/providers/<provider-id>.yaml`
- **Project**: `.0x0/providers/<provider-id>.yaml`

The filename (without `.yaml`) becomes the provider ID. These files use the same schema as the
`provider.<id>` config section and are merged at the appropriate precedence level.

## Setting the default model

```yaml
model: claude-code/claude-sonnet-4-6
```

The format is `provider-id/model-id`. This can be overridden per-agent or via the `--model` CLI
flag.

A separate `small_model` field sets the model used for lightweight tasks like title generation.
The small model is auto-detected from the active provider (prefers haiku/mini/nano variants):

```yaml
small_model: claude-code/claude-haiku-4-5-20251001
```

## Model selection priority

1. `--model` CLI flag
2. Agent-level `model` setting
3. `model` in config
4. Default: `claude-code/claude-sonnet-4-6` (if `claude` binary is on PATH)

Models are sorted by preference: sonnet > gpt-5-codex > opus > o3 > o4-mini > haiku.

## Listing models

```bash
# List all available models
0x0 models

# Filter by provider
0x0 models claude-code

# Verbose output with costs and metadata
0x0 models --verbose
```
