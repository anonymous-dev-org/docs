The `0x0.nvim` plugin provides a native Neovim chat experience — a floating window with streaming
responses, foldable tool output, `vim.ui.input` for prompts, and `vim.ui.select` for permissions.
No terminal buffer. Full vim motions and folds.

The plugin connects to the `0x0-server` daemon via HTTP and SSE for real-time message streaming,
permission dialogs, question dialogs, file reload events, and toast notifications.

Commands, agents, and skills are defined in your `config.yaml` and `.zeroxzero/` directory -- the
plugin fetches them from the server at runtime.

## Requirements

- Neovim >= 0.10
- `curl` in PATH
- [`0x0-server`](https://github.com/anonymous-dev-org/0x0) installed
- `ANTHROPIC_API_KEY` environment variable set (or configured in `config.yaml`)

## Installation

### lazy.nvim

```lua
{
  "anonymous-dev-org/0x0.nvim",
  opts = {},
}
```

### packer.nvim

```lua
use {
  "anonymous-dev-org/0x0.nvim",
  config = function()
    require("zeroxzero").setup()
  end,
}
```

## Configuration

All options and their defaults:

```lua
require("zeroxzero").setup({
  cmd = "0x0-server",             -- Server binary (falls back to "0x0 serve")
  port = 4096,                    -- Server port
  hostname = "127.0.0.1",
  auto_start = true,              -- Start server if not running
  chat = {
    width = 0.5,                  -- Fraction of editor width (0-1) or absolute columns (>1)
    height = 0.8,                 -- Fraction of editor height (0-1) or absolute rows (>1)
    border = "rounded",           -- Neovim border style
    fold_tools = true,            -- Auto-fold completed tool output
    show_thinking = false,        -- Show reasoning/thinking blocks
  },
  keymaps = {
    toggle = "<leader>0",
    context = "<leader>0f",
    session = "<leader>0s",
    interrupt = "<leader>0i",
    model = "<leader>0m",
    inline_edit = "<leader>0e",
  },
  auth = nil,                     -- Optional {username = "...", password = "..."}
})
```

## Keymaps

All keymaps are configurable via the `keymaps` table. Set any key to `""` (empty string) to disable it.

### Global

| Keymap | Mode | Action |
| --- | --- | --- |
| `<leader>0` | n | Toggle chat window |
| `<leader>0f` | n, v | Add file or visual selection to context |
| `<leader>0s` | n | Session picker |
| `<leader>0i` | n | Interrupt session |
| `<leader>0m` | n | Model picker |
| `<leader>0e` | n, v | Inline edit |

### Chat Buffer

| Key | Action |
| --- | --- |
| `q` | Close chat window |
| `<CR>` | New prompt |
| `<C-c>` | Interrupt current response |
| `<Tab>` | Cycle to next agent (per-prompt) |
| `<S-Tab>` | Cycle to previous agent (per-prompt) |
| `za` | Toggle fold (tool output) |
| `zM` | Fold all |
| `zR` | Unfold all |

## Commands

| Command | Description |
| --- | --- |
| `:ZeroToggle` | Toggle chat window visibility |
| `:ZeroContext` | Add current file (normal mode) or visual selection to context |
| `:ZeroSession` | Session picker |
| `:ZeroInterrupt` | Interrupt current session |
| `:ZeroModel` | Model picker |
| `:ZeroInlineEdit` | Inline edit at cursor position or visual selection |

## Chat Window

The chat window is a native Neovim floating buffer with:

- **Streaming responses** — text appears as the model generates it
- **Foldable tool output** — tool invocations fold into single lines with status icons (✔ success, ✘ error, ▶ running)
- **Permission dialogs** — `vim.ui.select` for allow/reject decisions
- **Question dialogs** — multi-step `vim.ui.select`/`vim.ui.input` sequences
- **Auto-scroll** — follows output unless you scroll up
- **Session switching** — load history from any session

### Chat Buffer Format

```
────────────────────────────────────────────────────
 You
────────────────────────────────────────────────────
fix the bug in auth.ts

────────────────────────────────────────────────────
 Assistant (claude-sonnet-4-6)
────────────────────────────────────────────────────

I'll fix the authentication bug.

✔ edit src/auth.ts [+5/-3]
│ @@ -10,5 +10,7 @@
│  function auth() {
│ -  return false
│ +  return true
│  }

✔ bash npm test
│ PASS src/auth.test.ts

The tests pass now.
```

Tool output lines prefixed with `│` are foldable. When `fold_tools` is enabled (default),
completed tool blocks auto-fold.

## Statusline

Integrate with lualine or any statusline plugin:

```lua
-- lualine example
sections = {
  lualine_x = {
    { function() return require("zeroxzero").statusline() end },
  },
}
```

Returns a string with connection status, animated spinner when busy, and current session title.
Returns empty string when not connected.

## How it works

1. **Connect** — Checks if `0x0-server` is running on port 4096. Starts it in the background if needed.
2. **Stream** — Opens SSE connection to `GET /event` for real-time message streaming.
3. **Prompt** — Sends messages via `POST /session/:id/prompt_async` (non-blocking).
4. **Render** — `message.part.updated` SSE events stream text deltas and tool results into the chat buffer.
5. **Interact** — Permission and question dialogs appear as native `vim.ui.select` overlays.

All requests include an `x-zeroxzero-directory` header so the server routes them to the correct
project instance. Buffers auto-reload when the agent edits files via SSE file-watcher events.

## Health check

Run the built-in health check to verify your setup:

```vim
:checkhealth zeroxzero
```

This checks Neovim version, `curl` availability, `0x0-server` binary presence, and server connectivity.
