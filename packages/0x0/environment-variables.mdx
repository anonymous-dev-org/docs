This page documents environment variables used by the current implementation.

## Runtime

- `ZEROXZERO_CLIENT`: client type identifier. Values: `app`, `cli`, `desktop`, `acp`. Controls which tools are available (e.g., `question` tool is only enabled for `app`, `cli`, and `desktop` clients).
- `ZEROXZERO_CALLER`: IDE identifier. Values: `vscode`, `vscode-insiders`. Used for VS Code integration detection.
- `ZEROXZERO_DISABLE_SHARE`: disable session sharing. Accepts `true` or `1`.
- `ZEROXZERO_API`: override API URL for session sharing.
- `ZEROXZERO_ROUTE`: internal TUI route state (used by the TUI package).
- `ZEROXZERO_TERMINAL`: set to `1` inside PTY subprocess shells spawned by the bash tool.

## Build-time

These are compile-time constants set during the build process:

- `ZEROXZERO_VERSION`: build version string.
- `ZEROXZERO_CHANNEL`: release channel.
- `ZEROXZERO_LIBC`: C library type for file watcher binary selection (`glibc` or `musl`, Linux only).

## Config interpolation

Config YAML files support `{env:VAR_NAME}` syntax to inject any environment variable value. This
means you can reference arbitrary env vars in your config without 0x0 needing to know about them
at compile time. See [Complete CLI Guide](/packages/0x0/complete-guide) for details.

## Notes

- Truthy boolean vars accept `true` or `1`.
- For config precedence details, see [Complete CLI Guide](/packages/0x0/complete-guide).
