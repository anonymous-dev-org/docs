0x0 is a lightweight HTTP server that provides two features:

1. **Inline code completions** -- context-aware, project-aware suggestions that get better the more you use them
2. **Git commit messages** -- generate conventional commit messages from staged diffs

It supports Claude (via the `claude` CLI) and OpenAI models (via `OPENAI_API_KEY`), runs on any machine with Bun, and compiles to a single self-contained binary.

## What makes it different

Most completion tools treat every request in isolation. 0x0 builds **three layers of context** for every completion:

- **Project conventions** -- scans your codebase once and detects formatting (semicolons, quotes, indentation), naming (camelCase, snake_case), import style, error handling patterns, and structural patterns (namespaces, async/await preference, export style). Cached per project.
- **File-level context** -- resolves imports from the current file, reads sibling files for patterns, greps for referenced type/interface definitions. Gathered per request.
- **Learned preferences** -- tracks which completions you accept and reject per project, automatically extracts recurring patterns into rules. The more you use it, the better it gets.

## Quick start

```bash
# Install
bun install -g @anonymous-dev/0x0

# Start the server (default port 4096)
0x0

# Or run from source
bun run src/index.ts
```

The server exposes a simple HTTP API. Your editor plugin sends `POST /completion` requests and receives streamed suggestions.

## Requirements

At least one LLM provider:

- **Claude** -- install the [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code) (`claude` binary on PATH)
- **OpenAI** -- set `OPENAI_API_KEY` environment variable
